# Тест-кейс: POS-AUTH-01 - Успешная авторизация

## Информация о тест-кейсе

| Параметр | Значение |
|----------|----------|
| **ID тест-кейса** | POS-AUTH-01 |
| **Название** | Успешная авторизация с действительным логином и паролем |
| **Приоритет** | Высокий |
| **Критичность** | Высокая |
| **Тип тестирования** | Функциональное |
| **Автор** | Raymundr |

## Предусловия

1. Браузер запущен и настроен
2. Открыта страница авторизации: `https://www.saucedemo.com/`
3. Сетевое соединение стабильное

## Тестовые данные

| Поле | Значение |
|------|----------|
| **Логин** | `standard_user` |
| **Пароль** | `secret_sauce` |

## Шаги выполнения

| № | Действие | Ожидаемый результат |
|---|----------|---------------------|
| 1 | В поле **"Username"** ввести значение `standard_user` | Поле заполнено текстом `standard_user`, курсор активен |
| 2 | В поле **"Password"** ввести значение `secret_sauce` | Поле заполнено маскированным текстом (•••••••••) |
| 3 | Нажать кнопку **"Login"** | Происходит перенаправление на страницу каталога продуктов |

## Ожидаемые результаты

После успешной авторизации система должна:

- [ ] Перенаправить на URL: `/inventory.html`
- [ ] Отобразить заголовок страницы **"Products"**
- [ ] Показать меню-бургер в левом верхнем углу
- [ ] Отобразить иконку корзины для покупок
- [ ] Показать список товаров (карточки продуктов)
- [ ] Отобразить фильтр для сортировки товаров
- [ ] Показать кнопку "Add to cart" для каждого товара

## Постусловия

- Пользователь аутентифицирован в системе
- Сессия пользователя активна
- Открыта главная страница приложения
- Доступны все функции авторизованного пользователя

# Тест-кейс: NEG-AUTH-01 - Негативные сценарии авторизации

## Информация о тест-кейсе

| Параметр | Значение |
|----------|----------|
| **ID тест-кейса** | NEG-AUTH-01 |
| **Название** | Негативные сценарии авторизации и проверка UI |
| **Приоритет** | Средний |
| **Критичность** | Средняя |
| **Тип тестирования** | Функциональное, UI, Валидация |
| **Автор** | Raymundr |

## Предусловия

1. Браузер запущен
2. Открыта страница авторизации: `https://www.saucedemo.com/`
3. Сетевое соединение стабильное

## Тестовые сценарии

### Сценарий 1: Авторизация заблокированного пользователя

**Тестовые данные:**
- **Логин:** `locked_out_user`
- **Пароль:** `secret_sauce`

**Шаги выполнения:**
1. Ввести логин `locked_out_user`
2. Ввести пароль `secret_sauce`
3. Нажать кнопку "Login"

**Ожидаемый результат:**
- [ ] Появляется сообщение об ошибке: "Epic sadface: Sorry, this user has been locked out."
- [ ] Сообщение об ошибке отображается в красном цвете
- [ ] Кнопка закрытия ошибки (×) отображается справа от сообщения
- [ ] Пользователь остается на странице авторизации
- [ ] Поля логина и пароля сохраняют введенные значения


![block](src/img/userblock.png)


### Сценарий 2: Неверный логин

**Тестовые данные:**
- **Логин:** `invalid_user`
- **Пароль:** `secret_sauce`

**Шаги выполнения:**
1. Ввести логин `invalid_user`
2. Ввести пароль `secret_sauce`
3. Нажать кнопку "Login"

**Ожидаемый результат:**
- [ ] Появляется сообщение об ошибке: "Epic sadface: Username and password do not match any user in this service"
- [ ] Сообщение об ошибке отображается в красном цвете
- [ ] Пароль отображается маскированным до и после ошибки
- [ ] Пользователь остается на странице авторизации
